FlashGenius — app walkthrough video script (~3 minutes)
Speak clearly; pause where marked. Have one short PDF or .docx ready before you record.

================================================================================
0:00–0:25  OPENING
================================================================================
“Hi, I’m [your name]. This is FlashGenius — my take on Problem One, the flashcard engine.

The idea: you drop in real notes as a PDF or Word doc, the app turns them into solid flashcards, and spaced repetition handles what you review and when — so you’re not just cramming once and forgetting.”

[PAUSE — show homepage or login for 2 seconds if you open cold]

================================================================================
0:25–0:55  LIBRARY
================================================================================
“After signing in, you land on the library. Up top you’ll see a greeting with your name once it’s set — you can tweak it with the little edit control next to the heading.

Below that are quick stats: how many decks you have, total cards, and what’s due today. Each row is a deck: tone preset, status, and actions — Book opens a reader on the source, All cards for editing the deck, Study now for a session, Generate if you need more cards, and Delete when you’re done with a deck.”

[SCROLL slowly through one deck row; hover a couple of buttons so the sage highlight shows]

================================================================================
0:55–1:45  NEW MATERIAL → CARDS
================================================================================
“I’ll use [New deck / quick upload — pick what you rehearsed] with a small file so this stays fast on video.

Upload pulls text on the server — keys never hit the browser. When the deck is ready, I hit Generate. Watch the cards stream in: that’s a two-step pipeline in the back — outline the chunk, then write typed cards — plus validation so junk doesn’t flood the deck.”

[DO the flow: upload OR new deck → show processing briefly → Generate → let 5–15 cards appear, then stop]

“If something’s off, I can jump into the card list and fix a front or back before I study.”

[OPTIONAL: open All cards, show one edit or just the list for 5 seconds — skip if tight on time]

================================================================================
1:45–2:25  STUDY
================================================================================
“For study I use SM‑2 — Again, Hard, Good, Easy. The app schedules the next time based on how well I knew the card, not just whether I flipped it.”

[Open Study now on a deck with cards — grade 2–3 cards, then exit or cut]

================================================================================
2:25–2:55  BOOK READER (IF YOU HAVE IT ON PROD)
================================================================================
“From Book I can read the source by page. If summaries are enabled, I can pull a short AI recap per page — it’s cached so I’m not burning tokens every time I revisit.”

[Open Book → one page → optional one summary — skip entire block if book isn’t set up; say: “Book view is in the repo; summaries need the deck_pages migration” in 5 seconds instead]

================================================================================
2:55–3:00  CLOSE
================================================================================
“That’s FlashGenius — live at [your URL], code on GitHub at [your repo]. Thanks for watching.”

================================================================================

[ ] At least one deck that already has cards OR time to generate on camera.

[ ] Turn off noisy notifications; mic test once.
